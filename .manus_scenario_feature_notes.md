Forward-looking scenario feature plan

Goal: Add a dedicated scenario JSON file path to the live portfolio adapter, normalize scenario data for the frontend, and render a preparation panel on the portfolio page.

Planned data additions:
- scenario source path in server config
- scenario payload with likely_next_action, readiness, message, conditions, trigger distances, and candidate asset
- website panel summarizing what to prepare for cold-storage rotations

Implementation approach:
1. Extend server/livePortfolio.ts with scenario file reading and fallback inference from existing strategy fields.
2. Expose normalized scenario object through livePortfolioRouter existing snapshot response.
3. Update client/src/pages/Portfolio.tsx to display a Preparation Panel above rankings/history.
4. Validate with TypeScript/build checks, then commit and prepare deployment steps.
